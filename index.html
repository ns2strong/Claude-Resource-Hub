  • AI analysts — Cowork + Claude Code (light) · mwe-claude-analysts

  • Microsoft 365 — Graph API, Teams, SharePoint · Phase 3
  • Internal APIs — .NET adapter, SQL Server · Phase 4
  • Azure DevOps — Boards, Repos, Pipelines · Phase 2
Add-now technologies
  • Python — AI pipelines, RAG, embeddings. Hire/upskill by Phase 2.
  • Node.js / TypeScript — MCP connector servers. Leverages existing TS.
  • Azure Container Apps — MCP hosting. Scale-to-zero, isolated per connector.
  • Azure AI Search — Hybrid RAG over document corpus. Phase 3.
Non-bypassable hooks
  • credential-scan — PostToolUse:Write, every file, every time
  • lint-on-write — PostToolUse:Write, ESLint/Black/dotnet format
  • pre-commit-gate — Blocks .env/.key files, enforces commit format
  • test-runner — PostToolUse:Write, related tests auto-run
Handoff non-negotiables
  • Every production solution: named owner AND deputy owner
  • Context files in Azure Repos before production registration
  • iManage OAuth re-authorised under new owner's identity on transfer
  • Old owner's tokens revoked within 24hr of departure
Test grader selection
  • Deterministic — API status, schema, token passthrough, credentials
  • LLM-graded — Brand voice, legal accuracy, completeness, disclaimer
  • Human review — Rubric calibration, P1 acceptance, post-regression
  • Default: deterministic. LLM only when deterministic is insufficient.
Platform principles
Identity-first

Every Claude action is traceable to an authenticated Azure AD user. No service accounts for matter-sensitive operations. The iManage connector passes the user's own OAuth token — Claude inherits their existing matter access and ethical walls.

Hooks are non-bypassable

Skills rely on Claude recognising context. Hooks fire deterministically at lifecycle events regardless. Put mandatory governance controls — credential scanning, lint gates, security validation — in hooks, not skills.

Document before deploying

If a Cowork solution's logic lives only in one person's session, it is in a pilot — not production. Context files in Azure Repos, a named deputy owner, and a Solution Registry entry are prerequisites for production status.

SharePoint for portals only

SPFx is excellent for dashboards and document viewers. It cannot host MCP servers, run background agents, or support streaming responses. Agentic solutions go to Azure Container Apps — not SharePoint.

Claude proposes, engineers decide

Claude Code operates in Plan mode for non-trivial changes. The deploy skill has disable-model-invocation: true — only a developer explicitly typing /deploy initiates a deployment.

Deterministic graders first

Use deterministic graders wherever possible. Use LLM graders where deterministic is insufficient. LLM judges must agree with human labels on ≥90% of a calibration set before they are trusted in CI.
